Rewarding the Scientific Process: Process-Level Reward Modeling for Agentic Data Analysis

四月 28, 2026 3-minute read
Paper-Digest • 2026-04-28

arXiv: 2604.24198 · PDF

作者: Zhisong Qiu, Shuofei Qiao, Kewei Xu, Yuqi Zhu, Lun Du, Ningyu Zhang, Huajun Chen

单位: Ant Group, Zhejiang University

主分类: cs.CL · 全部: cs.AI, cs.CE, cs.CL, cs.LG, cs.MA

命中关键词: large language model, llm, agent, agentic, reasoning, inference

TL;DR

DataPRM 是一个环境感知的生成式过程奖励模型，通过主动交互探测中间执行状态和三值反思奖励，解决通用 PRM 在数据分析 agent 上漏判静默错误、误罚探索行为的问题，仅 4B 参数即可在 ScienceAgentBench 上带来 7.21% 提升。

Motivation

Process Reward Model（PRM）在数学等静态域已经证明能显著增强 LLM 的推理能力，但搬到数据分析这种动态、需要与解释器反复交互的任务上就失灵了。作者做了一项实证研究，发现通用 PRM 存在两类系统性失败：一是漏掉 silent errors——代码没抛异常但逻辑错，结果数值错但 PRM 给高分；二是把必要的 trial-and-error exploration 当成 grounding 失败而过度惩罚，打压了 agent 正常的试错路径。

今天受这个问题困扰的是做 agentic data analysis 的团队——比如 ScienceAgentBench、DABStep、DABench、TableBench 这一类需要 agent 写 pandas/SQL、跑数据、再根据中间输出调整策略的场景。目前的 workaround 要么是只用 outcome reward（结果对就给分），粒度太粗、RL 时 credit assignment 很差；要么是直接套数学域 PRM，但静默错误和探索行为根本没进训练分布。

这件事现在才值得做的前置条件，是 agent 已经具备了调用解释器、观察中间状态的能力，PRM 本身也可以被设计成能主动与环境交互的 verifier，而不只是一个被动评分器。如果 PRM 不接入环境，就看不到那些只有执行后才暴露的 silent bug。作者的实际影响范围也说得比较诚实：聚焦 data analysis agent 这一类，不声称通用。

核心观点

  • 通用 PRM 在数据分析 agent 上存在两类系统性失败：漏判 silent errors、错罚探索动作。
  • PRM 需要成为 active verifier——可以自己调用环境去探中间状态，而不是只看 trajectory 文本打分。
  • 三值 reflection-aware 奖励：区分"可纠正的 grounding 错误"和"不可恢复的错误"，避免把探索当成失败。
  • 用 diversity-driven trajectory generation + knowledge-augmented step-level 标注，构造 8K+ 高质量训练样本。
  • 4B 参数就能打过更大基线，并且在 Best-of-N、Beam Search、DVTS、RL 四种下游用法中都稳定有效。

下图对应作者的 pilot study：通用 PRM 在 data analysis 上整体分数偏低，且在 grounding error 场景下的分数分布明显错配——本该低分的反而高分、探索步却被压到低分，直接佐证了上述两类失败模式。

方法

DataPRM 的定位是一个 环境感知生成式 PRM。给定历史 trajectory $h_t$ 和当前步 $\tau_t$，它不仅读文本，还具备工具调用能力主动重跑 / 探查中间执行状态，从而抓 silent error。奖励端采用 ternary reflection-aware 策略：正确 / 可纠正的 grounding 错误 / 不可恢复错误三档，让 agent 在探索时不被一刀切地扣分。

下图是它与 data analysis agent 协同的 pipeline：agent 负责执行任务，PRM 逐步监督每一步 procedural action，必要时主动和环境交互来验证。

训练数据方面，作者设计了 scalable 的两段式 pipeline：(a) diversity-driven trajectory generation 扩大行为覆盖 + knowledge-augmented step-level annotation 给每步打标；(b) 打分端用 multi-turn interaction、tool-augmented、reflection-aware 三种机制共同产出奖励。总共构造 8K+ 高质量训练实例。

实验

Benchmark：ScienceAgentBench、DABStep、DABench、TableBench。用法覆盖 Best-of-N、Beam Search、DVTS 等 Test-Time Scaling 策略，以及把 PRM 接进 RL 做 process reward 监督，与 outcome-reward baseline 对比。模型规模 4B。

结果

Best-of-N 推理下，DataPRM 在 ScienceAgentBench 上把下游 policy LLM 提升 7.21%，在 DABStep 上提升 11.28%。4B 参数跑赢更强的大尺寸 baseline。

在扩展的 TTS 策略下，Beam Search 和 DVTS 两种搜索、各计算预算档位上都稳定优于所有 baseline，说明它不是只在 Best-of-N 一种采样下 overfit。

接进 RL 后，process reward 相对 outcome-reward baseline 收益明显：DABench 达到 78.73%，TableBench 达到 64.84%，验证 step 级监督确实比只看最终答案更能稳住训练。

训练曲线也显示 reward 随 step 稳定上升，没出现 outcome-reward 常见的 reward hacking 塌陷。

环境交互这一组件的 ablation 证明主动探测中间状态是抓 silent error 的关键，去掉后性能回落。

结论

实践者最该带走的一点：给 data analysis agent 做 PRM，别只当被动文本评分器——让 PRM 自己能调环境去验证中间状态，并且用三值奖励区分"可纠正 grounding 错误"和"真错"，就能用 4B 的小模型把下游 policy 在 ScienceAgentBench 上推高 7.21%、DABStep 推高 11.28%，且在 Best-of-N / Beam Search / DVTS / RL 四种用法下都稳定受益。边界也要说清：结论集中在 data analysis 这一类可执行、可观测的任务；数学、开放域 reasoning 是否同样受益没有验证；训练数据规模只有 8K+，更大规模下的 scaling 行为仍是未知。

是否新瓶装旧酒

  • 对比 Math-Shepherd / PRM800K 这类静态域 PRM：核心 idea（step-level reward 监督）一致，但这些工作 PRM 是纯文本打分，没有环境交互；DataPRM 真正新的部分是让 PRM 变成 active verifier，主动跑中间状态抓 silent error。
  • 对比 code execution-based verifier（如用 unit test/解释器反馈做 reward）：也利用了执行信号，但执行信号一般只在 outcome 层或异常层；DataPRM 把它下沉到 step 级 PRM，并加入 ternary reflection-aware 以区分探索。
  • 对比 agent 的 self-reflection / self-critique：共享"区分可纠正错误"的直觉，但那一类是 actor 自评，DataPRM 是独立的 reward model 并用于 RL 训练。
  • 整体判断：不是换名字的增量工作，核心机制（环境交互 + 三值反思奖励）在 data analysis PRM 这个具体场景下是实打实的新设计，但概念级别的新颖度有限。

尚未回答的问题

  • PRM 主动调用环境带来的额外执行成本 / 延迟没有系统量化。
  • 8K 训练样本之外的 scaling 曲线未知，更大规模是否继续受益不确定。
  • 能否迁移到非数据分析的 agent 任务（如 web agent、软件工程 agent）未验证。
  • ternary reward 的阈值 / 判定是否对 prompt 分布敏感，缺更细的 ablation。
  • 与 outcome reward 混合使用的最优比例、以及在线 RL 中 PRM drift 问题没讨论。

原始摘要（中文翻译）

Process Reward Models（PRM）在数学等静态领域已经取得了显著成功，用于增强大语言模型（LLM）的推理能力。然而，它们在动态数据分析任务中的潜力仍未被充分探索。在这项工作中，我们首先给出一项实证研究，揭示通用领域的 PRM 难以监督数据分析 agent：具体而言，它们无法检测到 silent errors（那些不会触发解释器异常、但会产生错误结果的逻辑缺陷），并且会错误地惩罚探索性动作，把必要的 trial-and-error 探索误判为 grounding failure。为弥合这一 gap，我们提出 DataPRM，一种新颖的、环境感知的生成式过程奖励模型：(1) 可以作为 active verifier，自主地与环境交互以探测中间执行状态，揭示 silent errors；(2) 采用 reflection-aware 的三值奖励策略，区分可纠正的 grounding 错误与不可恢复的错误。我们设计了一个可扩展的 pipeline，通过 diversity-driven 的 trajectory 生成和 knowledge-augmented 的 step 级标注，为 DataPRM 构建了超过 8K 条高质量训练实例。实验结果表明，在 Best-of-N 推理下，DataPRM 在 ScienceAgentBench 上为下游 policy LLM 带来 7.21% 的提升，在 DABStep 上带来 11.28% 的提升。值得注意的是，DataPRM 仅用 4B 参数便超越了强基线，并在多种 Test-Time Scaling 策略下表现出稳健的泛化能力。此外，将 DataPRM 集成到强化学习中，相对于 outcome-reward 基线带来了显著收益，在 DABench 上达到 78.73%，在 TableBench 上达到 64.84%，验证了 process reward 监督的有效性。代码已公开在 https://github.com/zjunlp/DataMind 。
