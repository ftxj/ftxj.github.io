FlashOverlap: Minimizing Tail Latency in Communication Overlap for Distributed LLM Training

四月 28, 2026 3-minute read
Paper-Digest • 2026-04-28

arXiv: 2604.24013 · PDF

作者: Rezaul Karim, Austin Wen, Wang Zongzuo, Weiwei Zhang, Yang Liu, Walid Ahmed

单位: Toronto Ascend Team, Huawei

主分类: cs.LG · 全部: cs.CV, cs.DC, cs.LG

命中关键词: large language model, llm, inference, distributed training, parallelism, gpu, throughput, latency

TL;DR

FlashOverlap 用分解的 P2P 通信替代 reduce-scatter / all-gather collective，精确消除分布式 LLM 训练中通信-计算 overlap 的 tail latency，适配 TP、DP、TPSP、UP 等多种并行策略。

Motivation

LLM 规模膨胀迫使训练和推理不得不跨 GPU/TPU/NPU 切分，但 tensor 并行和 data 并行的 collective 通信开销已经成为算力利用率的主要瓶颈。当前业界的主流 workaround 是通信-计算 overlap，具体做法是把输入按 data slicing 切成若干 chunk、错峰调度——vLLM/Megatron/TransformerEngine 里都能看到这类实现。但作者指出一个具体的痛点：这些 data-slicing 方法会引入 tail latency——最后一个 chunk 没有可 overlap 的计算来掩盖，通信时间直接暴露在关键路径上，导致 overlap 的收益被末尾那一段吃掉。对于今天跑大规模分布式训练 / 长序列 TP 推理的团队来说，这意味着即便上了 overlap，MFU 仍然卡在一个次优点，而单纯堆带宽或者换 NIC 并不能解决这个尾部形状问题。作者认为现在值得做的原因是：ring 型 collective 的内部步骤本来就是一串 P2P，只要敢把 collective 拆开并重新调度 partitioned compute，就能把尾部那一刀填掉。本文的作用范围明确——收紧到通信 overlap 的 tail，不追求改网络拓扑，也不碰 collective 语义。

核心观点

  • 指出现有 data-slicing overlap 方法共同的 tail latency 问题，是 overlap 增益天花板的主因。
  • 提出 Flash-Overlap：用分解后的 P2P 通信取代 reduce-scatter / all-gather 这两个 collective。
  • 给出一个精确（exact）算法调度 partitioned 计算，使通信与计算细粒度重叠，尾延迟被消除而非掩盖。
  • 方案通用：兼容 data-parallel、TP、TPSP、UP 等主流并行策略，不是某一种并行的一次性补丁。
  • 实验声称同时拿到更低 latency、更高 MFU 和更高 throughput。

上图展示的就是作者要消除的问题：TP 下 attention 用未 overlap 的 reduce-scatter，通信时间直接堆在计算后面形成明显的 overhead——这正是 data-slicing 方案在最后一个 chunk 也解决不掉的 tail。

方法

核心思路是把 ring 型 collective 拆回它底层的 P2P step 序列，并把与之对应的 partitioned 计算重排，让每一个 P2P step 都有一段"自己这一份"的 partial compute 来掩盖。具体包括两个对称的构件：

  • Flash All-Gather：按 ring 顺序把 AG 拆成 N-1 个 P2P send/recv，每一步完成后立刻对已到达的分片启动对应的 partial 计算，下一轮 P2P 与当前 partial compute 异步重叠。
  • Flash Reduce-Scatter：对称地把 RS 分解为 P2P 步 + partial 归约，partitioned compute 与 P2P 交错发射。

上面两张图给出 AG 与 RS 的语义起止状态，是理解为什么可以被拆的前提：两者本质上都是分片在 rank 间的一个重新排布 / 规约模式，作者正是要把这个排布过程打开。

这两张图进一步展示 AG / RS 的标准 ring 实现——四 rank 逐步传递，每一步只和邻居 P2P。这说明 collective 内部本来就是 P2P 序列，Flash-Overlap 的拆解并不打破语义，只是把隐式的 step 显式暴露出来供调度。

这是方法的关键落地图：四 rank 下的 Flash All-Gather / Flash Reduce-Scatter，把 ring 顺序的 P2P step 与 partial-output 计算交错，使每个通信 step 都被一段算力覆盖，末尾不再出现只通信不计算的空洞。作者强调这是一个 exact 算法——不是启发式切 chunk。

把上述 Flash-RS 套进 TP attention，就得到 Query-Split Flash-RS Attention：Q 维切分后用异步分解 P2P 和 attention 计算重叠，对应消除图 7 中那段暴露的 RS overhead。

实验

摘要层面给出的对比维度包括：latency、Model FLOPS Utilization (MFU)、throughput；覆盖 data-parallel 训练以及 TPSP、UP 在内的多种 tensor 级并行。摘要未披露具体模型、序列长度、集群规模、基线系统名称与数值。

结果

作者报告 Flash-Overlap 在上述设置下"一致地（consistently）“拿到更低延迟、更高 MFU 和更高吞吐。由于摘要没给数字，相对 data-slicing overlap 的量化增益、在什么规模下增益最大、通信/计算比例分别是多少这些关键点都待正文确认。

结论

实践者带走的 single takeaway：如果你的 distributed LLM 训练/推理已经上了 data-slicing 的 comm-compute overlap 但 MFU 仍卡住，瓶颈很可能是尾部那一段暴露的 collective，而 Flash-Overlap 通过把 RS/AG 拆成 P2P + partitioned compute 精确调度，从机制上而非启发式地消除它。最能支撑这一点的是作者给出的 “exact algorithm that eliminates tail latency” 这一定性主张以及 MFU/throughput 同向改善的报告。边界要说清楚：摘要未提供具体数字、未公开集群规模与基线，也没描述 ablation，因此"能把 tail 压到多小"“在非 ring 拓扑或异构带宽下是否仍然 exact"仍需正文与复现验证。

是否新瓶装旧酒

倾向真·增量工作，但改进维度相对聚焦：

  • CoCoNet / T3 / Megatron 的 SP-overlap：同样把 collective 与 compute 重叠，但多数是 chunk-level data slicing，正是本文指出会产生 tail latency 的一类；FlashOverlap 的差别在于 collective 被分解成 P2P 后一次性消掉尾部，而不是增加 chunk 数量渐近逼近。
  • Flux / TransformerEngine 的 TP overlap kernel：同样面向 TP 的 AG/RS 融合，但大多以 kernel fusion / stream 级别编排为主；本文更强调调度算法层的 exact 重排，与 kernel 层实现正交。
  • SplitAttention 类工作：把 attention 在 Q 维切分以便重叠，这在本文 Query-Split Flash-RS 里也有影子——相似之处是 Q-split 思路，新点在于把 Q-split 直接挂到分解 P2P 的 RS 调度上。

总体上不确定性在于摘要未公布具体 baseline 名称，真正的新颖度需要看正文对 CoCoNet/T3/Flux 的直接对比。

尚未回答的问题

  • 消除 tail latency 的精确增益数字：MFU 提升多少、throughput 提升多少？
  • 在非 ring 拓扑（如 rail-optimized、NVSwitch all-to-all）下 exact 性是否仍成立？
  • 长序列 + 低带宽（跨节点 InfiniBand）下收益与近节点 NVLink 的差异？
  • 与 kernel-level overlap（Flux 类）叠加是否还有边际收益？
  • 推理侧（prefill vs decode）的适用性与对 TTFT / ITL 的影响未说明。
  • 分解 P2P 对调度器复杂度、对 NCCL/HCCL 栈的侵入程度、对容错的影响。

原始摘要（中文翻译）

大语言模型规模的快速增长要求把计算负载切分到 GPU、TPU、NPU 等加速器上执行。然而，这些并行策略会带来大量的数据通信开销，严重拖累计算效率。虽然通信-计算 overlap 是一个有前景的方向，但现有基于 data slicing 的方案存在 tail latency 问题。为克服这一限制，本研究提出一种新颖的通信-计算 overlap 技术，用于消除当前最先进 overlap 方法在分布式 LLM 训练中的 tail latency。该技术的目标是有效缓解分布式训练与推理中 tensor parallelism 和 data parallelism 的通信瓶颈。具体而言，我们提出一种名为 Flash-Overlap 的新方法：用分解式的点对点（P2P）通信替代传统的 reduce-scatter 和 all-gather collective 操作，并调度分片计算以实现细粒度 overlap。我们的方法提供了一个用于降低通信开销并消除 tail latency 的 exact 算法。此外，它提供了一个通用解决方案，兼容 data-parallel 训练以及包括 TPSP 和 UP 在内的多种 tensor-level parallelism 策略。实验评估表明，我们的技术在延迟、Model FLOPS Utilization (MFU) 和吞吐等方面都一致取得更优结果。
